# Smarter Tools - sx信息化项目智能监控与分析平台

针对 **sx政府采购网** 的全流程数据智能化解决方案。集成了**自动采集**、**语义筛选**、**移动端监控**以及**投标文件智能比对**等核心功能，旨在提升招投标工作的信息获取效率与文档审核准确性。

## ✨ 核心模块

### 1. 📊 智能监控日报 (Monitor)
全自动化的招投标信息采集引擎，专注于**信息化建设**领域。
- **语义精准筛选**：内置 BAAI 语义向量模型，智能识别“软件开发”、“系统集成”、“数据服务”等高价值项目，自动过滤非相关采购（如“办公耗材”）。
- **多维度数据提取**：深度解析公告源码，提取**预算金额**、**开标地点**、**采购方式**、**代理机构**等并规范化输出。
- **智能去重清洗**：自动识别“更正公告”与“终止公告”，并基于项目编号逻辑自动剔除已失效的原始公告，确保日报清单准确无误。
- **多端可视**：
    - 💻 **PC 端大屏** (`/`)：提供日历视图、项目聚合统计、Excel 一键导出。
    - 📱 **移动端** (`/mobile`)：卡片式流式布局，专为手机浏览优化，随时随地掌握动态。
    - 📦 **采集控制台** (`/caiji`)：可视化的任务调度与日志监控。

### 2. ⚖️ 投标文件智能比对 (Bid Comparator)
基于 AI 与规则引擎的文档一致性校验工具，防止**围标串标**与**低级错误**。
- **多格式支持**：支持 PDF、Word (.docx, .doc) 文件的文本提取与混合比对。
- **智能排除逻辑**：
    - **招标文件排除**：自动识别并排除招标文件中的通用条款（如“投标人须知”），避免误报。此过程支持**模糊排除**（90%相似度），可忽略因 OCR 或标点导致的微小差异。
- **深度风险检测**：
    - **疑似修改/拼写错误**：检测除标点外的内容微小篡改或错别字（如“报告”写成“宝告”）。
    - **共同差异/错别字 (⭐核心亮点)**：当两份标书内容**完全一致**时，系统会额外比对招标文件。如果发现两份标书**同时**与招标文件存在相同的细微偏差（如都把“文件”写成了“蚊件”），将标记为**“疑似共同修改”**（强串标特征）。
    - **敏感信息核查**：自动提取并比对文中出现的手机号、身份证、邮箱等实体信息。
    - **序号逻辑校验**：检查文档中的序号（1, 2, 3...）是否存在中断或错误（如 1->3），同时智能忽略大跨度跳跃（如 1->8，视为新条款）。

### 3. 🗄️ 归档与工具箱
- **文件归档**：自动归档历史比对文件，支持重命名备注与版本管理。
- **工具箱 (`/bai`)**：集成常用办公提效小工具的下载分发。

## 🛠️ 技术栈

- **Core**: Python 3.9, Flask
- **Data & AI**: 
    - `sentence-transformers` (语义分析)
    - `scikit-learn` (向量计算)
    - `pdfplumber`, `python-docx` (文档解析)
    - `difflib` (差异比对算法)
- **Frontend**: HTML5, Vanilla JS, CSS3 (Glassmorphism 玻璃拟态设计)
- **Infra**: Docker, APScheduler (定时任务)

## 🚀 快速开始

### 方式一：Docker 部署 (推荐)

该项目提供开箱即用的 Docker 支持。

```bash
# 构建镜像
docker build -t smarter-tools .

# 运行容器 (映射 5000 端口)
docker run -d -p 5000:5000 smarter-tools
```

### 方式二：本地运行

1. **环境准备**
   确保 Python 3.8+ 环境。

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **启动服务**
   ```bash
   python run.py
   ```
